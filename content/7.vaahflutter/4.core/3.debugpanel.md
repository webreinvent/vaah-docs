- This panel uses EnvironmentConfig, thus depending on the env.dart file.

Debug panel will be visible only if `showDebugPanel` is set to `true` for `EnvironmentConfig` in the `env.dart` file. You can change the color of the tag by setting the `debugPanelColor` variable for your EnvironmentConfig.

::alert{type="danger" class="flex items-center p-4 mb-4 text-sm text-red-800 rounded-lg bg-red-50 dark:bg-gray-800 dark:text-red-400" role="alert"}
Note:   
Remember showDebugPanel for production should always be `false` in the Environment configuration in the `env.dart` file.
::

```dart
  'production': defaultConfig.copyWith(
    ...
    showDebugPanel: false,
  ),
```

## Make the panel visible on a specific screen

```dart
final _navigatorKey = GlobalKey<NavigatorState>();

class MyPage extends StatelessWidget {
  const MyPage({Key? key}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return DebugWidget(
        navigatorKey: _navigatorKey,
        child: Container(),
    );
  }
}
```

## Make the panel visible on every screen using central implementation.

::alert{type="info" class="flex items-center p-4 mb-4 text-sm text-blue-800 rounded-lg bg-blue-50 dark:bg-gray-800 dark:text-blue-400" role="alert"}
NOTE:   
You have to write the below code in MaterialApp, and that will show the tag panel on each screen. You don't have to wrap any other screen/ widget, or you don't have to extend any screen/widget with DebugWidget.
::

In the file containing the material app paste this code after imports
```dart
final _navigatorKey = GlobalKey<NavigatorState>();
```

 In the material app paste, this code and panel will be visible on all pages.
```dart
return MaterialApp(
  builder: (BuildContext context, Widget? child) {
    return DebugWidget(
      navigatorKey: _navigatorKey,
      child: child!,
    );
  },
);
```

## Add custom section to the debug panel

### About Custom Section
Custom Section is a widget responsible for displaying custom data when a developer wants to display some data not natively included in the debug panel or environment configuration. 

Before adding a custom section, there are three components to understand in the custom section widget. 

#### **`custom_debug_sections`**
- **Type**: Array of Objects
- **Description**: This is the main array that contains all the custom sections to be added to the debug panel.

**Array Elements**

Each element in the `custom_debug_sections` array represents a section in the debug panel.

**Section Object**
- `section_name`

  - **Type**: String
  - **Description**: The name of the section to be displayed in the debug panel.
- `content_holder`

  - **Type**: Object
  - **Description**: Holds the `content` for the section in a structured format.

  - `content`
    - **Type**: Object
    - **Description**: Contains the key-value pairs that will be displayed as a table within the section.
    - `Key (e.g., "Test Topic 1")`
      - **Type**: String
      - **Description**: Represents the topic or the row heading in the table.
    - `Value`
        - **Type**: Object

        - **Description**: The value object holds the actual content for the topic.

        - `value`
          - **Type**: String
          - **Description**: The value corresponding to the topic which will be displayed in the table.
        - `color`
          - **Type**: int
          - **Description**: The color code for the value. If not provided, the default color will be `warning`.

### To display the custom data on the debug panel just add the data in the json file as show in the below snippet.

Add you data to any of the environment configuration json files; `develop.json`, `staging.json`, `production.json` or any custom json file you have created for the environment configuration. 

For more information about `enviroment configuration` [click here](../3.essentials/2.environments.md).

- You have to add a key in the json file i.e., `"custom_debug_sections"` with list of maps containing section information as shown below.

```json
{
  ...Other Content

  "custom_debug_sections": [
    {
      "section_name": "Test Section 1",
      "content_holder": {
        "content": {
          "Test Topic 1": {
            "value": "Test Value 1"
          }
        }
      }
    },
    {
      "section_name": "Test Section 1",
      "content_holder": {
        "content": {
          "Test Topic 1": {
            "value": "Test Value 1"
          },
          "Test Topic 2": {
            "value": "Test Value 1"
          }
        }
      }
    },
    {
      "section_name": "Test Section 1",
      "content_holder": {
        "content": {
          "Test Topic 1": {
            "value": "Test Value 1"
          }
        }
      }
    }
  ]
}

```
